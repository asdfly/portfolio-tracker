五、数据一致性修复

问题描述

2026-04-27 的日收益数据显示为 +10.03%（日盈亏 +62,088元），与实际市场行情严重不符。

5.1 根因分析

数据表数据来源4/27 数值
portfolio_snapshots.market_value通达信持仓文件（静态）673,023.40
portfolio_summary.total_value新浪实时行情 API680,849.60

差额 +7,827元 导致 daily_return 被错误计算。通达信持仓文件未更新（仍为 4/24 数据），而新浪 API 返回的是 4/27 实时价格。两个数据源的时间戳不一致。

5.2 修复方案（3 处）

修复1：database.py save_portfolio_snapshot

优先使用新浪实时行情数据填充 snapshot 的 current_price、market_value、pnl 字段，与 summary 数据源统一。

current_price = holding.get('realtime_price', holding.get('current_price')) market_value = holding.get('realtime_market_value', holding.get('market_value')) pnl = holding.get('realtime_pnl', holding.get('pnl'))

修复2：portfolio.py _calculate_summary

prev_value 改用新浪 pre_close × quantity 反推前日市值，覆盖全部持仓（不再排除 daily_change_pct=0 的记录）。

修复3：portfolio.py _update_realtime_quotes

  • 新增价格偏离校验：实时价格与当前价格偏离超过 ±30% 时拒绝更新
  • 新增无行情 fallback：当新浪 API 未返回某 ETF 行情时，使用通达信静态数据填充

九、已知局限与后续方向

9.1 当前局限

#局限影响建议
1通达信持仓文件需手动导出数据更新依赖人工操作配置定时任务自动执行通达信导出脚本
2新浪API存在频率限制高频调用可能被限流已实现 retry + delay 机制，可考虑接入东方财富作为备用源
3Brinson 归因基准权重为估算归因精度受限引入真实指数成分权重数据
4Monte Carlo 假设正态分布极端行情预测不准可改用历史模拟法或 GARCH 模型
5通知系统默认关闭无法实时接收告警配置企业微信 Webhook 或邮箱 SMTP

9.2 后续方向

  1. 自动化调度：配置 APScheduler 定时任务，交易日 15:30 自动执行分析并推送报告
  2. 多账户支持：扩展数据库模型，支持多个投资组合并行管理
  3. 实盘交易接口：对接券商API，实现信号驱动的半自动交易
  4. 回测框架增强：支持自定义策略回测、因子分析、组合优化
  5. 移动端适配：Streamlit 响应式布局优化，支持手机/平板访问
