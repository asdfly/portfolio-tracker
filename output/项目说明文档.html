1   项目概述

投资组合跟踪分析系统是一套完整的 ETF 投资组合管理与智能分析工具。系统从通达信交易终端导出持仓数据，自动获取实时行情，计算技术指标与风险指标，生成可视化报告，并提供 Web 交互式 Dashboard。

核心能力

  • 持仓跟踪 — 23 只 ETF 实时行情获取、市值/盈亏自动计算
  • 技术分析 — MA / MACD / RSI / KDJ / 布林带 / ATR 六大指标全覆盖
  • 风险分析 — 夏普比率、最大回撤、年化波动率、VaR、Beta
  • 基准对比 — 11 个主流指数实时对比（沪深300、中证500 等）
  • 智能建议 — 基于动量/估值/风险的自动再平衡建议
  • 可视化 — Streamlit Dashboard，净值曲线、收益分布、回撤曲线、持仓饼图
  • 自动报告 — HTML 增强版报告，含图表，可邮件推送
  • 告警监控 — 跌幅/回撤/集中度/波动率异常自动告警

4   数据库设计

系统使用 SQLite 数据库 data/database/portfolio.db，共 8 张核心表。

4.1 portfolio_snapshots（持仓快照）

记录每个交易日每只 ETF 的持仓明细。

字段类型说明

dateTEXT交易日期 (YYYY-MM-DD)
codeTEXT证券代码（如 sh510300）
nameTEXT证券名称
quantityREAL持仓数量
cost_priceREAL成本价
current_priceREAL当前价格
market_valueREAL最新市值
pnlREAL持仓盈亏
pnl_rateREAL盈亏率 (%)
ytd_returnREAL年初至今收益率 (%)
betaREAL贝塔系数

4.2 portfolio_summary（组合汇总）

记录每个交易日的组合整体指标，含风险指标。

字段类型说明

total_costREAL总成本

4.6 数据库索引

Dashboard 启动时自动创建以下索引以加速查询：

idx_snap_date          ON portfolio_snapshots(date)
idx_snap_code_date     ON portfolio_snapshots(code, date)
idx_summary_date       ON portfolio_summary(date)
idx_idx_quote_code_date ON index_quotes(code, date)
idx_tech_date          ON etf_technical(date)
idx_tech_code_date     ON etf_technical(code, date)

6   核心功能详解

6.1 四阶段分析流程（run_analysis.py）

阶段功能说明
阶段一基础分析读取通达信持仓 → 新浪/东方财富获取行情 → 计算技术指标 → 写入数据库
阶段二风险分析计算夏普比率、最大回撤、年化波动率、VaR
阶段三监控告警检测日跌幅/回撤/集中度/波动率异常，触发邮件或企业微信通知
阶段四智能分析生成投资建议（动量/估值/风险），策略回测，输出 Markdown 智能报告

6.2 数据源

系统使用多数据源策略，优先级如下：

优先级数据源接口用途
1新浪财经hq.sinajs.cn实时行情、K线数据
2东方财富push2.eastmoney.com备用实时行情
3AKSharePython SDK补充数据
网络说明：东方财富 API（push2his.eastmoney.com）在当前网络环境下可能被阻断，历史回填已切换至新浪财经 K 线接口（money.finance.sina.com.cn）作为主数据源，腾讯财经作为备用。

6.3 持仓数据导入

持仓数据从通达信交易终端导出，格式为制表符分隔的 .xls 文件。

  1. 在通达信中打开持仓窗口

  3. 保存到 C:\zd_zsone\T0002\export\持仓股YYYYMMDD.xls

7.2 性能优化

  • @st.cache_data — 所有数据库查询函数带 5 分钟缓存，相同参数命中零延迟
  • 降采样 — 图表数据自动压缩至 500 个点，保留首尾边界值不失真
  • SQLite 索引 — 6 个复合索引加速常用查询
  • 缓存预热 — 页面加载时后台静默预生成最近 10 个交易日 + 6 个时间预设的缓存
  • end_date 参数 — load_summary / load_index_quotes 支持按选定日期截断，切换历史日期时返回该日期视角的数据

7.3 重启 Dashboard

8   数据更新流程

8.1 日常更新（推荐方式）

  1. 在通达信中导出最新持仓文件到 C:\zd_zsone\T0002\export\
  2. 修改 config/settings.py 中的 POSITION_FILE 路径指向新文件
  3. 运行分析：
    python run_analysis.py
  4. 刷新 Dashboard 页面查看最新数据
周末执行：系统默认跳过周末，如需周末执行，使用以下命令绕过交易日检查： 
python -c "import run_analysis; run_analysis.is_trading_day = lambda: True; exit(run_analysis.main())"

8.2 历史数据回填

若需补充历史数据（如首次部署或新增 ETF），使用回填脚本：

python backfill_full_history.py                  # 回填全部（ETF + 指数）
python backfill_full_history.py --etf-only        # 仅 ETF
python backfill_full_history.py --index-only      # 仅指数
python backfill_full_history.py --dry-run         # 预览模式，不实际写入

回填数据源为新浪财经 K 线接口，备用腾讯财经接口。

9   定时任务配置

scheduled_run.bat 为定时任务入口脚本，通过 Windows 任务计划程序配置自动执行。

配置步骤

  1. 打开「任务计划程序」（taskschd.msc）
  2. 创建基本任务，设置触发器为「每个工作日 15:30」
  3. 操作选择「启动程序」，指向 scheduled_run.bat
  4. 确保 POSITION_FILE 指向最新的通达信持仓文件
前提：每日 15:30 前需从通达信导出持仓文件并更新配置路径。如持仓未变化，分析仍会运行（使用数据库中的现有持仓 + 新行情）。

11   常见问题

Q1: Dashboard 打开报 Connection Error

Streamlit 进程可能已退出。执行 taskkill /F /IM streamlit.exe 清理后重新运行 start_dashboard.bat。

Q2: 选择长时间范围（2年+）时 Dashboard 响应慢

已通过缓存 + 降采样优化。首次加载长周期数据时会有短暂延迟（后台预热），后续切换时间范围会命中缓存，响应瞬间完成。

Q3: 东方财富 API 连接失败

当前网络环境下东方财富 API 可能被阻断。系统已自动切换至新浪财经接口作为主数据源。历史回填脚本同时内置了腾讯财经作为备用接口。

Q4: 如何添加新的 ETF 持仓

  1. 在通达信中添加新 ETF 到持仓
  2. 导出新的持仓 .xls 文件
  3. 更新 config/settings.py 中的 POSITION_FILE 和 MAJOR_ETFS
  4. 运行 python backfill_full_history.py 回填新 ETF 的历史数据
  5. 运行 python run_analysis.py 更新当天数据

Q5: 数据库文件过大如何处理

可定期清理 daily_news 表的历史新闻数据（DELETE FROM daily_news WHERE date < 'YYYY-MM-DD'）。核心行情数据建议保留以维持风险指标准确性。

Q6: 如何配置邮件通知

运行 python setup_notification.py 启动配置向导，按提示填写 SMTP 服务器、账号、授权码、收件人等信息。配置保存在 config/settings.py 的 NOTIFICATION_CONFIG 中。
