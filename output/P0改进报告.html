一、改进背景与目标

Dashboard 基线版本已具备净值走势、持仓分布、风险概览和收益日历四个核心标签页。然而与 QuantStats、pyfolio 等专业工具对标后，发现缺少以下关键分析维度：

  • 风险调整收益指标不完整（仅有夏普比率，缺少 Sortino、Calmar）
  • 缺乏月度收益的结构化视图（热力图）
  • 缺少动态风险指标的时间序列展示（滚动夏普、滚动波动率）
  • 存在 Streamlit 弃用警告和周末脏数据等代码质量问题
  • 缓存策略未充分优化，日期选择器受 LIMIT 限制

P0 改进目标：在不引入新数据源的前提下，用最小改动量补齐专业级投资组合分析工具必备的核心功能。

二、改进项清单

共 7 项改进，全部已完成交付。

2.1 扩展风险指标

在 compute_extended_risk_metrics() 函数（L210-297）中新增 5 项专业风险指标，基于 portfolio_summary 表的全部历史日收益率计算。

指标 计算公式 参数 解读
Sortino 比率 年化收益 / 下行标准差 无风险利率 2.5% 衡量单位下行风险的超额收益，数值越高越好。与夏普比率相比，仅惩罚下行波动。
Calmar 比率 年化收益 / 最大回撤绝对值 年化交易日 252 天 衡量单位最大回撤的年化收益，特别适合评估趋势策略表现。
胜率 盈利天数 / 总交易天数 × 100% 日收益率 = 0 视为平局，不计入分母 反映策略的盈利稳定性。高胜率不等于高收益，需结合盈亏比综合判断。
盈亏比 平均盈利金额 / 平均亏损金额 按日盈亏金额计算 反映策略在盈利时的平均获利幅度相对亏损时的平均亏损幅度。
最大回撤持续天数 净值低于历史最高值的连续最长交易日数 遍历净值序列 衡量最坏情况下的恢复时间。与最大回撤幅度配合使用，评估策略的抗压能力。
设计决策：Sortino 和 Calmar 的计算复用了已有的年化收益率和最大回撤逻辑，仅新增了下行标准差和持续天数的计算，代码增量约 50 行。胜率和盈亏比直接从日收益率序列统计，无需额外参数。

2.2 月度收益热力图

新增 compute_monthly_returns() 函数（L301-323），将每日收益率聚合为年份 × 月份的二维矩阵。

设计要素说明
数据源portfolio_summary 表的 date 和 daily_return 字段
聚合逻辑按 (年份, 月份) 分组，对 daily_return 求和（即月度累计收益率）
矩阵结构行 = 年份，列 = 1-12 月，末尾附加年度合计列
可视化Plotly go.Heatmap，RdYlGn 色阶，绿色 = 正收益，红色 = 负收益
数值标注热力图每个格子内显示百分比数值，保留 2 位小数
UI 位置Tab3 风险分析底部，L1678
参考对标：pyfolio 的月度收益热力图是业界标准呈现方式。我们的实现额外增加了年度合计列和汇总行，方便快速评估年度表现。

2.3 滚动夏普比率和滚动波动率

新增 compute_rolling_metrics() 函数（L325-349），计算动态风险指标的时序变化。

设计要素说明
数据源portfolio_summary 表的 daily_return 全量历史
滚动窗口默认 60 个交易日（约 3 个月），与侧边栏时间范围联动
滚动夏普rolling_mean / rolling_std × sqrt(252)，减去无风险利率
滚动波动率rolling_std × sqrt(252)，年化处理
可视化双子图折线图，上方蓝色夏普曲线（正区域填充），下方灰色波动率曲线
UI 位置Tab1 净值走势，基准对比图下方，L1155
交互设计：用户切换侧边栏时间范围时，滚动窗口大小保持 60 日不变（固定窗口更便于观察趋势变化），但数据截取范围随时间范围联动。

2.4 净值曲线基准归一化

  • 组合净值和基准指数统一归一化到 100 基点：nav = current_value / base_value × 100
  • 基准基准日期为选定时间范围的起始日
  • 图中同时显示组合净值线和基准净值线，便于直观比较超额收益
  • 图例显示累计收益率百分比
