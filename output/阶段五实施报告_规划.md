# 投资组合跟踪系统 - 阶段五实施计划

**规划日期**: 2026年5月10日  
**当前版本**: v1.8  
**目标版本**: v2.0  
**阶段**: 专业版功能扩展

---

## 一、阶段目标

在前五个阶段（基础架构/风险分析/自动化部署/智能分析/Dashboard可视化）的基础上，进一步提升系统的专业性和实用价值，向专业投资分析工具看齐。Phase 5 聚焦以下五个方向：

1. **自定义指标工作台** — 让用户构建和验证自己的技术指标
2. **资金流分析** — 追踪市场资金动向，辅助择时决策
3. **数据导出增强** — 生成专业的 Excel/PDF 分析报告
4. **组合归因深化** — 引入多因子模型和风格暴露分析
5. **系统体验优化** — 性能、布局、多组合支持

## 二、模块详细规划

### 2.1 自定义指标工作台 (Dashboard 新 Tab)

**定位**: Tab 9 — 自定义分析

#### 核心功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 指标公式编辑器 | 支持用户通过下拉菜单+参数输入构建技术指标组合 | P0 |
| 指标信号回测 | 对自定义指标进行历史信号回测，统计胜率/盈亏比 | P0 |
| K线形态识别 | 内置常见K线形态（十字星/锤子线/吞没/启明星等） | P1 |
| 指标模板库 | 预置常用指标组合模板（如"均线金叉+MACD多头"） | P1 |
| 自定义信号告警 | 将自定义指标信号接入告警引擎 | P2 |

#### 预置指标模板

| 模板名称 | 公式逻辑 | 适用场景 |
|----------|----------|----------|
| 均线多头排列 | MA5 > MA10 > MA20 > MA60 | 趋势确认 |
| MACD金叉 + RSI未超买 | MACD金叉且 RSI < 70 | 买入信号 |
| 布林带下轨反弹 | 价格触及布林下轨后回升 | 超跌反弹 |
| 量价背离 | 价格创新高但成交量递减 | 顶部预警 |
| 多空分歧度 | 涨跌家数比 + 涨停跌停比 | 市场情绪 |

#### 技术方案

- 在 `dashboard.py` 中新增 Tab 9
- 指标公式使用 JSON 配置存储（不执行用户代码，保障安全）
- 信号回测复用现有 `backtest.py` 的回测框架
- K线形态识别添加到 `src/analysis/technical.py`

#### 数据库变更

```sql
-- 自定义指标表
CREATE TABLE custom_indicators (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    formula TEXT,          -- JSON 格式的指标定义
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_template BOOLEAN DEFAULT 0
);

-- 指标回测结果表
CREATE TABLE indicator_backtest_results (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    indicator_id INTEGER,
    test_period TEXT,      -- 测试区间
    total_signals INTEGER,
    win_count INTEGER,
    loss_count INTEGER,
    win_rate REAL,
    avg_pnl REAL,
    sharpe REAL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (indicator_id) REFERENCES custom_indicators(id)
);
```

#### 预估工作量

- Dashboard Tab 9: ~350 行
- 技术形态识别: ~200 行
- 指标回测逻辑: ~150 行
- 总计: ~700 行代码

### 2.2 资金流分析 (Dashboard 新 Tab)

**定位**: Tab 10 — 资金动向

#### 核心功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 行业资金流向图 | 展示各行业近N日资金净流入/流出 | P0 |
| ETF资金流向 | 单只ETF的资金流入流出趋势 | P0 |
| 北向资金关联 | 沪深港通北向资金与持仓关联分析 | P1 |
| 主力/大单追踪 | 大单净买入统计，异常大单预警 | P2 |

#### 数据来源

| 数据项 | 来源 | 获取方式 |
|--------|------|----------|
| ETF份额变动 | 基金公司公告 | AKShare `fund_etf_fund_daily` |
| 北向资金 | 东方财富 | AKShare `stock_hsgt_north_net_flow_in_em` |
| 行业资金流 | 东方财富 | AKShare `stock_sector_fund_flow_rank` |
| 大单数据 | 东方财富 | AKShare `stock_individual_fund_flow` |

#### 可视化组件

- **行业资金热力图**: 横轴日期、纵轴行业、颜色深浅表示净流入金额
- **ETF资金流趋势图**: 面积图展示净流入/流出累积值
- **资金流排行**: 各行业/ETF资金净流入排行榜
- **资金异常预警**: 当单日资金流向出现异常波动时触发告警

#### 数据库变更

```sql
-- 资金流向数据表
CREATE TABLE fund_flows (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date TEXT NOT NULL,
    code TEXT,             -- ETF代码或行业代码
    name TEXT,
    net_inflow REAL,       -- 净流入(万元)
    buy_amount REAL,       -- 买入额
    sell_amount REAL,      -- 卖出额
    category TEXT,         -- 'etf' / 'sector' / 'north'
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 预估工作量

- 数据采集: ~300 行 (新增 `src/data_sources/fund_flow.py`)
- Dashboard Tab 10: ~400 行
- 数据回填: ~100 行
- 总计: ~800 行代码

### 2.4 组合归因深化

**定位**: Tab 3 风险分析的扩展

#### 2.4.1 多因子归因 (Fama-French)

| 因子 | 说明 | 数据源 |
|------|------|--------|
| 市场因子 (Rm-Rf) | 市场超额收益 | 沪深300指数收益率 |
| 规模因子 (SMB) | 小盘股超额收益 | 中证1000 - 沪深300 |
| 价值因子 (HML) | 价值股超额收益 | 红利指数 - 成长指数 |
| 动量因子 (MOM) | 动量效应 | 过去12月收益率排序 |
| 质量因子 (QMJ) | 盈利质量 | ROE/利润率指标 |

#### 2.4.2 风格暴露分析

- **规模维度**: 大盘 / 中盘 / 小盘暴露度
- **风格维度**: 价值 / 成长 / 均衡暴露度
- **行业维度**: 相对基准的行业超配/低配
- **雷达图可视化**: 多维风格暴露一目了然

#### 2.4.3 行业轮动分析

- 近N日/月/年各行业收益排名变化
- 行业轮动速度指标（排名变化的标准差）
- 资金流与行业表现关联度

#### 技术方案

- 因子数据使用公开指数收益率构造（无需额外数据源）
- 时间序列回归（OLS）计算因子暴露度
- 风格暴露使用持仓ETF的行业/规模/风格标签加权

#### 预估工作量

- 因子归因计算: ~250 行 (新增 `src/analysis/factor_attribution.py`)
- Dashboard 展示: ~300 行
- 总计: ~550 行代码

### 2.5 系统体验优化

#### 2.5.1 性能优化

| 问题 | 方案 | 预期效果 |
|------|------|----------|
| 大数据量图表渲染慢 | Plotly `max_points` 降采样（已有）+ WebAssembly 加速 | 渲染提速 2-3x |
| 数据库查询慢 | 添加复合索引 + 查询结果缓存（已有 `@st.cache_data`） | 查询提速 3-5x |
| 页面首次加载慢 | 数据预加载 + Skeleton 占位 | 感知提速 |
| 重复计算 | 全局变量缓存中间结果 | CPU 减少 30% |

**数据库索引优化**:
```sql
CREATE INDEX IF NOT EXISTS idx_snapshots_date_code ON portfolio_snapshots(date, code);
CREATE INDEX IF NOT EXISTS idx_summary_date ON portfolio_summary(date);
CREATE INDEX IF NOT EXISTS idx_quotes_date_code ON index_quotes(date, code);
CREATE INDEX IF NOT EXISTS idx_technical_date_code ON etf_technical(date, code);
```

#### 2.5.2 响应式布局

- 主布局适配 1280px ~ 2560px 屏幕宽度
- 关键指标卡片在小屏幕上自动换行
- 图表高度根据容器宽度自适应

#### 2.5.3 数据刷新策略

| 数据类型 | 刷新频率 | 策略 |
|----------|----------|------|
| 当日行情 | 盘中每5分钟 | 自动刷新按钮 + 手动刷新 |
| 技术指标 | 每日收盘后 | 定时任务更新 |
| 新闻资讯 | 每30分钟 | 后台异步抓取 |
| 基准指数 | 盘中每5分钟 | 与行情同步 |

#### 2.5.4 多组合支持 (v2.0+)

- 支持创建多个投资组合（如"A股组合"/"可转债组合"）
- 组合间对比分析
- 合并视图查看全部持仓
- 数据库层面通过 `portfolio_id` 字段区分

#### 预估工作量

- 性能优化: ~200 行 (索引 + 缓存策略)
- 布局优化: ~100 行 (CSS + 布局调整)
- 数据刷新: ~150 行 (自动刷新机制)
- 总计: ~450 行代码

## 三、实施优先级与时间线

### 分批实施策略

| 批次 | 模块 | 预计代码量 | 建议版本 |
|------|------|-----------|----------|
| **Batch 1** | 2.3 数据导出增强 + 2.5 系统体验优化 | ~1,300 行 | v1.9 |

| **Batch 4** | 2.2 资金流分析 | ~800 行 | v2.0 |

### 优先级排序理由

1. **数据导出增强 (Batch 1)**: 实用性最高，直接提升日常使用效率
2. **系统体验优化 (Batch 1)**: 基础性能保障，后续功能的前提
3. **组合归因深化 (Batch 2)**: 专业分析能力提升，数据源现成
4. **自定义指标工作台 (Batch 3)**: 差异化功能，开发量适中
5. **资金流分析 (Batch 4)**: 依赖外部数据采集，实现复杂度最高

## 四、技术风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| AKShare API 不稳定 | 资金流数据获取失败 | 多数据源备选 + 本地缓存 |
| Plotly 图表过多导致性能下降 | 页面加载卡顿 | Tab 懒加载 + 图表虚拟化 |
| PDF 生成依赖库冲突 | 环境兼容性问题 | 使用容器隔离或降级为 HTML |
| 多组合支持需要数据库迁移 | 数据一致性风险 | 迁移脚本 + 数据备份 |
| 自定义指标公式安全性 | 潜在注入风险 | 严格限制为 JSON 配置，不执行任意代码 |
